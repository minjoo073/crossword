# K-pop Album Crossword — 디자인 시스템

> 현재 활성 디자인 스타일: **#1 — Crowny Class** (출처: `.claude/knowledge/ui-designer/styles/01-crowny-class.md`)
> 이 프로젝트는 Crowny Class 토큰 시스템을 **베이스**로 쓰되, 게임 화면에 한해 **앨범별 accent**를 오버라이드합니다.
> radius·shadow·spacing·폰트 스케일·8px 그리드는 **그대로 유지**, 색만 갈아끼웁니다 (스타일 §18 규칙).

---

## 0. 사용자 목표 (UX 먼저)

K-pop 팬이 **좋아하는 아티스트 앨범을 고르고 → 크로스워드를 풀고 → 완성 인증**을 짧고 기분좋게.
타깃: 4·5세대 팬, **모바일 90%**. 핵심 감정 = "내 최애 세계관 안에서 노는 느낌".

### 메인 플로우 (단계 ≤3, 분기 ≤2)

```
[1] 앨범 고르기            [2] 퍼즐 풀기                 [3] 완성·공유
홈 그리드 →               칸 탭 → 키패드 입력 →         100% 채움 →
앨범 카드 탭              가로/세로 단서 읽기            컨페티 + 공유 카드
   │                         │                              │
   │ 이탈: 최애가 안보임      │ 이탈: 어디 입력중인지 모름   │ 이탈: 공유 동선 없음
   │ → 검색/성별 필터        │ → 활성 단어 하이라이트       │ → 결과 카드 즉시 노출
   │                         │ → 틀린 칸 즉시 피드백        │
   └ 측정: 카드 탭률          └ 측정: 첫칸까지 시간/정답률   └ 측정: 완성률·공유 클릭
```

이탈 3대 원인 = (1) 최애를 못 찾음 (2) 지금 어느 단서를 푸는지 모름 (3) 틀린 줄 모르고 헤맴.
→ 각각 **검색·성별필터**, **활성 단어 줄 하이라이트**, **즉시 오답 피드백**으로 해소.

## 1. 브랜드 / 시각 위계

- **앱 셸(홈·헤더·탭바)** = Crowny Class 기본 **보라→분홍 그라데이션**. K-pop 팝 무드와 그대로 맞아 유지.
- **게임 화면** = 진입한 앨범의 `accent`/`bg`/`fg`로 갈아탐 → 화면 자체가 "그 앨범 세계관".
- 첫째로 보이는 것 → **그라데이션/accent CTA** ("풀기 시작", "정답 확인").
  둘째 → **활성 단어 줄 + 활성 칸**. 셋째 → 단서 텍스트·번호 라벨.

### 에디토리얼 강조 (CEO 취향 — 박스·글라스 금지)

- AI티 나는 **둥근 박스 카드 남발 금지**, **반투명 글라스(backdrop blur) 금지**.
- 강조는 **큰 타이포 + 색면 + 얇은 1px 구분선 + 여백**으로. 카드는 그림자 약하게(`--shadow-sm`/`md`)만.
- 앨범 컨셉은 **컬러·타이포·그래픽 도형**(라인, 점, 대형 숫자/한글자)으로 표현 — 사진 없이.

## 2. 컬러 토큰

### 2.1 앱 셸 (Crowny 기본 그대로)

| 토큰 | HEX | 용도 |
|---|---|---|
| `--primary` | `#8A38F5` | 셸 CTA·링크·활성 |
| `--secondary` | `#D53A6B` | 그라데이션 끝 |
| `--gradient` | `linear-gradient(135deg,#8A38F5,#D53A6B)` | 홈 CTA·로고·완성 배지 |
| `--light-bg` | `#F3F4F6` | 홈 페이지 배경 |
| `--text-dark` | `#1F2937` | 라이트 본문 |

상태색(성공 `#22C55E` / 에러 `#EF4444` / 정보 `#3B82F6`)도 Crowny 그대로 사용.

### 2.2 앨범 테마 오버라이드 (게임 화면 한정)

게임 화면 루트에 `data-album="aespa-whiplash"` 같은 속성을 주고, JS로 `album-themes.json` 값을 CSS 변수에 주입:

```css
/* 게임 화면 스코프에서만 */
.game[data-themed] {
  --accent:    var(--album-accent-1);  /* 정답·CTA·활성 칸 테두리 */
  --accent-2:  var(--album-accent-2);  /* 보조 하이라이트·세로줄 */
  --accent-3:  var(--album-accent-3);  /* 포인트(번호·아이콘) */
  --album-bg:  /* JSON bg */;           /* 화면 배경 */
  --album-fg:  /* JSON fg */;           /* 배경 위 텍스트 */
}
```

- `accent[0]` = 주 강조(정답/CTA/활성 테두리), `accent[1]` = 보조(세로 단어 줄), `accent[2]` = 포인트.
- **bg가 어두운 앨범**(aespa·LE SSERAFIM·PLAVE·IVE·BABYMONSTER 등)은 다크 테마, **밝은 앨범**(RIIZE·ZB1·TWS·ILLIT·BND·NCT WISH)은 라이트 테마로 자동 분기 (`fg` 명도로 판별).
- 명도 대비 **WCAG AA(본문 4.5:1)** 유지하도록 bg/fg를 미리 짝지어 둠.

## 3. 타이포그래피

### 폰트 전략 (혼용 금지 — 딱 3개)

- **본문/UI 전부 = Pretendard Variable** (한국어·영문·게임 UI 모두). 기본값.
- **레트로 창 크롬만 = Tahoma** — `.win__title`(타이틀바)·`.win__menu`(메뉴)·`.taskbar`/`.start-btn` 등 XP 셸 한정. 그 외 어디에도 Tahoma 금지.
- **로고만 = Agbalumo** — `.brand`("K-POP CROSSWORD"). 본문·제목·UI엔 절대 안 씀.
- 드림코어 라벨(`--dc-font`)도 **Pretendard** (이전엔 Tahoma로 새던 버그 — 교정함).

### 굵기 — 딱 3단계

`--weight-regular: 400` / `--weight-semibold: 600` / `--weight-bold: 800`.
500·700·900 난무 금지. (예외: XP 타이틀바 700은 크롬 폰트와 함께 의도적 스큐어모픽 예외.)

- 400 = 본문·단서 텍스트
- 600 = 라벨·버튼·강조·칸 글자·키패드·메타
- 800 = 앨범 타이틀·섹션 헤딩·대형 그래픽·CLEAR!

### 타입 스케일 (압축 — CEO 승인 오버라이드)

> 글로벌 Crowny의 16px 하한을 **이 게임 화면 밀도**를 위해 완화. **UI 텍스트 하한 = 13px**, 그 아래로는 안 내림. (칸 번호 12px만 크로스워드 관례상 예외.)

| 토큰 | px | 용도 |
|---|---|---|
| `--font-xs` | 13px | 메타·태그·작은 라벨 (하한) |
| `--font-sm` | 14px | 보조 텍스트·컴팩트 버튼·툴·타이머 |
| `--font-md` | 15px | 본문·단서 텍스트 (기본) |
| `--font-lg` | 16px | 활성 단서·기본 버튼 |
| `--font-xl` | 18px | 키패드 키·서브헤딩 |
| `--font-2xl` | 22px | 앨범 타이틀 |
| `--font-3xl` | 28px | 완성 CLEAR! 타이틀 |

칸 안 글자는 `clamp(15px, 4.2vw, 20px)`, 칸 번호는 12px(오버레이 숫자, 관례 예외).

## 4. 모서리 / 그림자 / 간격 (Crowny 그대로)

- radius: 칸 `--radius-sm`(6px), 버튼 `--radius-md`(10px), 카드 `--radius-xl`(16px).
- 그림자: 카드 `--shadow-sm`~`md` 까지만(에디토리얼 — 무겁게 X). 완성 모달만 `--shadow-xl`.
- 간격: 8px 그리드. 칸 사이 gap 2px, 격자 외곽 패딩 `--spacing-md`(16px).
- 터치 타겟 **44×44px** 이상 (칸·키패드 키).

---

## 5. 라이트 / 다크

- **앱 셸** = 라이트 기본. 다크모드는 셸 토큰(`--light-bg`→다크)만 스왑.
- **게임 화면**은 앨범 `bg`/`fg`가 곧 테마라 OS 다크모드와 **독립** — 앨범 무드가 우선.
- 다크 앨범 화면에선 셸 헤더/탭바도 해당 `bg`에 맞춰 어둡게 톤다운(투명 글라스 X, 불투명 색면).

## 6. 크로스워드 격자 비주얼 스펙

### 6.1 칸(Cell) 상태 — 5종 필수

| 상태 | 배경 | 테두리 | 글자 | 비고 |
|---|---|---|---|---|
| **빈칸(입력 가능)** | `--album-bg` 위 카드색(라이트=흰, 다크=`fg` 8% 색면) | `1px` `--album-fg` 20% | 없음 | 기본 |
| **차단칸(non-cell)** | 투명/없음 | 없음 | — | 격자 빈 공간, 렌더 안 함 |
| **채워진칸** | 위와 동일 | 동일 | `--album-fg`, 20~22px, weight 700 | 입력된 글자 |
| **활성칸(커서)** | `--accent` 18% 색면 | `2px solid --accent` | `--accent` 또는 fg | 지금 입력 위치 |
| **활성 단어 줄** | `--accent-2` 12% 색면 | — | — | 커서가 속한 가로/세로 전체 |
| **정답칸(확정)** | `--success` 14% 또는 `--accent` 14% | — | `--success`/fg | 단어 완성 시 |
| **오답칸** | `--error` 14% (`#FEE2E2` 류) | `1px --error` | `--error` | 확인 시 또는 즉시 |
| **disabled/검수중** | fg 6% | 없음 | fg 40% | 채점 중 잠금 |

- **번호 라벨**: 칸 좌상단, **16px가 부담이면 칸을 키워서라도 12~13px은 금지** → 칸 글자 22px 기준 라벨은 14px 예외 허용하되 **최소 13px 절대 하한**(가독 위해 칸 자체를 충분히 크게: 모바일 한 변 ≥ 36px, 권장 40px). 색 = `--accent-3`, weight 600.
- **가로(Across)/세로(Down) 하이라이트**: 활성 단어 방향에 따라 줄 전체 `--accent-2` 12% 깔고, 활성칸만 `--accent` 강조. 방향 토글 = 같은 칸 두 번 탭.

### 6.2 완성 연출

- 단어 1개 완성 → 그 줄 칸들 `--accent`로 **순차 pop**(0.04s stagger, scale 1→1.08→1).
- 퍼즐 100% → **컨페티**(앨범 `accent` 3색) + 중앙 완성 카드(Pretendard 800 "CLEAR!") + 공유 버튼(그라데이션).
- 컨페티/모션은 `prefers-reduced-motion` 존중 — 끄면 색 페이드만.

### 6.3 모바일 칸→키패드 UX (권장안)

```
┌─────────────────────────┐
│  [앨범 타이틀 · 진행 n/12] │  ← 상단 고정, 불투명 색면(글라스 X)
│                         │
│     ┌─┬─┬─┬─┬─┐         │  ← 격자: 화면폭에 맞춰 정사각, 한 변 ≥40px
│     │1│ │ │ │2│         │     스크롤 없이 보이게 auto-fit
│     ├─┼─┼─┼─┼─┤         │
│     │ │█│ │ │ │         │  █ = 활성칸
│     └─┴─┴─┴─┴─┘         │
│                         │
│  ┌─ 활성 단서 바 ──────┐  │  ← 격자 바로 아래 고정, 항상 보임
│  │ 3�across  "그룹 리더"│  │     ◀ ▶ 로 단어 이동
│  └─────────────────────┘  │
│   Q W E R T Y U I O P    │  ← 자체 A-Z 키패드(OS 자판 X,
│    A S D F G H J K L     │     영문 대문자만·숫자/특수 제거)
│     Z X C V B N M  ⌫     │     키 44×44, 입력 시 다음칸 자동 이동
└─────────────────────────┘
```

- **OS 키보드 대신 커스텀 A-Z 키패드** 권장: 답이 전부 영문 대문자(quizzes.json `answerRules`)라 자판 흔들림·자동완성·줌 방지. 키 = 44px, 활성 칸 색과 동일 accent로 눌림 표시.
- 칸 탭 → 활성칸 지정 + 단서 바가 그 단어로 갱신. 글자 입력 → 같은 단어 다음 빈칸으로 자동 전진, 단어 끝나면 정지(자동 점프 금지 — 혼란).
- ⌫ = 현재칸 비우고 한 칸 후진. 단서 바 ◀▶ = 이전/다음 단어로 점프.
- **states**: 키패드 입력중(default)/채점중(disabled, 회색)/완성(키패드 숨김, 공유 CTA로 교체).

### 6.4 단서 표현

- 풀세로 단서 목록은 **접이식 시트**가 아니라, 격자 아래 **활성 단서 바 1줄**을 기본으로(에디토리얼·집중). 전체 목록은 "단서 전체 보기" 텍스트 링크 → 별도 화면/하단 패널(불투명).
- Across/Down은 Lucide `arrow-right` / `arrow-down` 아이콘 + 번호로 구분.

### 저작권 가드

- **공식 앨범 사진 번들 금지.** 이미지가 들어갈 자리는 `<div class="album-art-slot">` 슬롯으로 비워두고, 그 안을 **그라데이션 색면 + 대형 한글자/숫자 타이포(Pretendard 800) + 라인 그래픽**으로 채워 컨셉을 표현.

## 8. 강개발 핸드오프 (픽셀 아닌 토큰·상태)

- 게임 화면 루트에 `data-album` → JS가 `album-themes.json`에서 `--accent/-2/-3`, `--album-bg`, `--album-fg` 주입. 나머지 토큰은 Crowny `tokens.css` 그대로 `var()` 참조.
- 칸 컴포넌트는 **상태 클래스**로: `.cell`(빈) `.cell--filled` `.cell--active` `.cell--in-word` `.cell--correct` `.cell--error` `.cell--locked`. 색은 전부 `var(--accent…)`/상태색.
- **빠뜨리면 안 되는 상태**: 키패드 `disabled`(채점중), 격자 `empty`(앨범 첫 진입), `error`(오답칸·로드 실패), `loading`(테마 주입 전 스켈레톤 색면), `focus`(키보드 탭 시 활성칸 포커스 링 `0 0 0 3px accent 30%`).
- 모션은 `prefers-reduced-motion` 분기 필수.
- CSS 라이브러리 금지(순수 CSS), 임의 HEX 금지(앨범색도 JSON→변수 경유), 폰트 Pretendard+Agbalumo, 아이콘 Lucide.
